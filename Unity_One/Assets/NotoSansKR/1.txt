🎨 0단계: 캔버스(Canvas) 설정 (도화지 준비)
Hierarchy(계층) 창 우클릭 ➡ UI ➡ Canvas 클릭.

새로 생긴 Canvas를 클릭하고 Inspector(속성) 창을 보세요.

Canvas Scaler 컴포넌트에서:

UI Scale Mode를 Scale With Screen Size로 변경.

Reference Resolution을 1920 x 1080으로 설정. (이렇게 해야 화면 크기가 바뀌어도 UI가 안 깨집니다.)

📜 1단계: 방 목록이 들어갈 '스크롤 뷰' 만들기
방이 100개가 되면 스크롤을 내려야 하니까, 목록 상자를 먼저 만듭니다.

Canvas 우클릭 ➡ UI ➡ Scroll View 생성.

이름을 **LobbyScrollView**로 바꾸세요.

화면 가운데 큼직하게 크기를 키우세요.

[중요] LobbyScrollView 왼쪽 화살표(▶)를 눌러서 계층을 펼치세요.

LobbyScrollView ➡ Viewport ➡ Content 라는 녀석을 찾으세요.

Content 오브젝트를 클릭하고, Inspector 창에서 Add Component 클릭.

Vertical Layout Group 검색해서 추가.

(기능: 자식들을 세로로 예쁘게 줄 세워줌)

Control Child Size의 Width를 체크하세요. (가로폭 꽉 차게)

Content Size Fitter 검색해서 추가.

Vertical Fit을 Preferred Size로 변경.

(기능: 목록이 늘어나면 스크롤 영역도 같이 길어지게 함)

🎫 2단계: 방 목록 아이템(Template) 만들기
목록에 들어갈 "한 줄짜리 디자인"을 만듭니다.

아까 찾은 Content 오브젝트 우클릭 ➡ UI ➡ Image 생성.

이름을 **LobbyItemTemplate**으로 바꾸세요.

색상은 흰색 말고 약간 회색으로 해서 눈에 띄게 하세요.

LobbyItemTemplate 안에 내용물을 넣습니다. (우클릭 ➡ UI ➡ Text - TextMeshPro 등)

Text (TMP) 1개: 이름 RoomNameText (방 제목 표시용)

Text (TMP) 1개: 이름 PlayerCountText (1/4 표시용)

Button (TMP) 1개: 이름 JoinButton (참가 버튼)

[스크립트 연결]

**LobbyItemTemplate**을 클릭.

작성해둔 LobbyListSingleUI.cs 스크립트를 드래그해서 넣으세요.

스크립트 칸에 빈칸이 3개 보이죠?

Lobby Name Text 칸에 ➡ 아까 만든 RoomNameText 드래그.

Players Text 칸에 ➡ 아까 만든 PlayerCountText 드래그.

Join Button 칸에 ➡ 아까 만든 JoinButton 드래그.
